Pathfinding Algorithms Overview

  • Lo-Temp Team
  • December 10th, 2023

We thought it would be helpful to explain how our pathfinding and communication link checkpoint selection works.

For pathfinding, we use Dijkstra's Algorithm and Theta*. These algorithms operate on graphs consisting of nodes and edges and allow us to find the optimal routes between points under some minimization criteria. We use Dijkstra's Algorithm to minimize the amount of hill-climbing done. To use Dijkstra’s Algorithm, we first take our graph and perform a preprocessing step known as Node Splitting on it such that all information is contained solely in the edges. Each edge stores the slope between its two nodes, so we can then apply Dijkstra’s algorithm to find the path that minimizes the cumulative weight—the path with the least hill climbing.

We use Theta* to minimize the distance between two locations while adhering to the maximum slope restriction. Theta* is a variant of the more well-known algorithm A* that allows for something known as any-angle path planning. Any-angle path planning takes advantage of the fact that the grid of nodes is really sampled from an underlying continuous surface and so allows us to travel directly between two nodes assuming there is a “line-of-sight” to that node, reducing the travel distance compared to if we were forced to travel along the grid lines. This line-of-sight check makes sure that there are no “obstacles” (locations on the grid that we cannot travel through) between two nodes, ensuring the validity of the path. In our case, the obstacles correspond to nodes whose slopes are greater than the maximum threshold.

To select the optimal communication link checkpoints, we divide up the path into eleven segments of equal length. This allows us to find the ideal case checkpoints which are optimally spaced along the path. We then check whether each of these points is a valid location for a checkpoint (whether its elevation angle to the earth is greater than its elevation angle to the horizon). We do this check with accurate data provided by the SPICE toolkit. If any points are invalid locations, we move their locations along the path, check whether they are now valid, and iterate until all checkpoints are at valid locations. We may then adjust the other points if possible to attempt a more even spread of the checkpoints. This minimizes the deviation from the ideal case and allows us to space out the points optimally.
